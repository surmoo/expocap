# ExpoCapture — деплой на GitHub Pages

## Один раз, занимает 5 минут

### 1. Создай аккаунт на github.com (если нет)

### 2. Создай новый репозиторий
- Нажми "+" → "New repository"
- Название: `expocap` (или любое)
- Visibility: Public
- Нажми "Create repository"

### 3. Загрузи файлы
- Нажми "uploading an existing file"
- Перетащи все три файла: index.html, sw.js, manifest.json
- Нажми "Commit changes"

### 4. Включи GitHub Pages
- Settings → Pages
- Source: "Deploy from a branch"
- Branch: main / (root)
- Нажми Save

### 5. Готово
Через 1-2 минуты приложение будет доступно по адресу:
https://ИМЯ_АККАУНТА.github.io/expocap/

Открой этот адрес на телефоне → Safari/Chrome спросит разрешение на микрофон → разреши.

## Добавить на домашний экран

**iOS (Safari):**
Кнопка "Поделиться" → "На экран Домой"

**Android (Chrome):**
Меню "⋮" → "Добавить на главный экран"

## Работа офлайн
При первом открытии с интернетом приложение кэшируется.
Дальше работает без сети — даже в подвале выставочного центра.

## Что происходит с файлами
Каждое фото и аудио СРАЗУ скачивается в папку Downloads на телефоне.
Закрыл приложение — файлы уже сохранены. Ничего не теряется.

Имя файла содержит всю структуру:
